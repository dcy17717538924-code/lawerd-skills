## 匹配逻辑

1. 从用户指令提取文书类型关键词
2. 按上表匹配 → 命中则加载 `.template.docx`
3. 未命中 → 走自由起草路径（从 Agent知识库 参考范文）
4. 多个模板可组合为"套装"（如撤诉+退费、诉状+保全）

## 套装

> 部分文书有固定搭配关系——某个业务场景下通常需要同时出具多份文书。
> 套装规则描述了这种关系：触发条件、包含的模板、共享信息、执行顺序。

### 已有套装

| 套装名 | 包含模板 | 触发场景 |
|--------|---------|---------|
| 撤诉退费 | 撤诉申请书 + 诉讼费退费申请书 | 庭外和解、撤回起诉 |
| 起诉+保全 | 诉状 + 诉讼保全申请书 | 财产类案件立案 |

### 套装工作流

当识别到触发场景时，按以下流程处理：

1. **识别套装** — 从律师指令中提取关键词（如"撤诉""退费""和解"）
2. **加载全部模板** — 一次性加载套装中所有 `.template.docx`
3. **共享信息提取** — 案号、法院、当事人信息等一次追问，填入所有模板
4. **逐一确认** — 每份文书单独展示确认，律师确认一张出一张
5. **联动提示** — 生成第一份时提示"该场景还需出具：[关联文书名]"

### 如何新增套装

在案件管理中遇到新场景 → 提取文书组合 → 在此表新增一行 → 同时确保模板仓库中有对应 `.template.docx`。
