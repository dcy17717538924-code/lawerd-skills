# 起草法律文书流程

执行本技能前，务必严格遵守 `WorkStandards` 中的红线与原则。

## 强制流程

### 步骤0：知识库搜索（自动执行，不打断流程）
- 收到起草指令后，在追问确认的同时，后台搜索 `D:\Agent知识库`：
  - 按**文书类型**定位对应子目录
  - 按**案由关键词**匹配文件名
  - 读 1-3 篇最接近的既往文书
- **不等待搜索结果**——搜索和追问并行。参照范文的结构和句式风格生成初稿。
- 搜索方式：优先用 Obsidian CLI `search`，失败则降级为 `search_content`。

### 步骤1：确认文书类型 + 查模板库
- 收到指令后首复："收到起草 [文书类型] 的指令。"
- **模板匹配**：对照 [[template-matching]] 表，确认文书类型是否有对应 `.template.docx`。
  - **有模板** → 回复："已匹配模板 [模板名]，将按模板填充关键信息。"
  - **无模板** → 回复："未找到模板，将参照知识库范文自由起草。"
- 追问确认核心诉求、涉案主体及文书类型。

### 步骤1.5：信息映射（依据 [[placeholder-dictionary]]）
- **有模板路径**：
  1. 加载 `.template.docx`，提取所有 `[占位符]`
  2. 对照 [[placeholder-dictionary]] 分类列出需提供的信息
  3. ⚡ 必填项 → 硬 Hook 追问律师
  4. ○ 选填项 → 有默认值自动填，无默认值跳过
  5. △ 条件项 → 根据案件类型判断是否追问
- **无模板路径**：
  1. 从知识库范文提取结构要素
  2. 用 [[placeholder-dictionary]] 生成追问清单
  3. 持续追问直至框架清楚

### 步骤2：初稿生成与确认
- **有模板路径**：占位符反向填充真实信息 → 生成初稿
- **无模板路径**：参照知识库范文风格自由起草
- 遵循{{USER_SHORT_NAME}}风格：简洁利落，当事人信息一段到底，诉讼请求顶格排列，事实与理由一段到底
- 在对话中直接展示初稿全文。确认后进入步骤3。

### 步骤3：DOCX 定稿（OfficeCLI 模板填充 + 格式预览 + 审阅）

> **核心原则：模板即格式权威。只替换文本，不触碰格式。**

#### 3a. 有模板路径（主路径）

1. **复制模板**到案卷目录：
   ```
   copy "{{REASONIX_SKILLS}}draft-legal-docs\templates\[模板名].template.docx" "[案卷目录]\[案件名]-[文书类型]（定稿）.docx"
   ```

2. **读取模板结构**——用 OfficeCLI 获取段落/表格路径和占位符位置：
   ```
   "{{HOME_WIN}}\AppData\Local\OfficeCLI\officecli.exe" view "[定稿路径].docx" text
   ```
   - 要素式表格模板 → 记录单元格路径 `/body/tbl[1]/tr[N]/tc[1或2]`
   - 传统段落模板 → 记录段落路径 `/body/p[N]`
   - **注意**：合并单元格（colspan=2）只暴露 `tc[1]`，需用 `tc[1]` 而非 `tc[2]`

3. **构建 batch JSON**——每条命令写一个 `{"command":"set","path":"...","props":{"text":"..."}}`：
   - 只替换需要改的段落/单元格，不动标题、空行、分隔线
   - 传统段落模板特别注意：
     - 列表编号段落的文本**不带序号前缀**（模板样式自动加编号）
     - 多余段落设 `"text":""` 清空；段落不足时模板多出的行设空处理
     - 模板段落数与本方案件不匹配时，优先保证已填充内容正确，多余空编号行属可接受瑕疵
   - 要素式表格模板：直接填值，多余行留空即无痕迹

4. **执行 batch 填充**（一次性写入，零格式损耗）：
   ```
   "{{HOME_WIN}}\AppData\Local\OfficeCLI\officecli.exe" batch "[定稿路径].docx" --input "[同目录]/batch.json" --json
   ```
   全部 `succeeded` 才通过；任一 `failed` 需排查路径后重试。

5. **⛔ 格式预览（强制）**——渲染 HTML 检查排版：
   ```
   "{{HOME_WIN}}\AppData\Local\OfficeCLI\officecli.exe" view "[定稿路径].docx" html -o "[同目录]/[文件名]-preview.html"
   ```
   检查项：标题字体（黑体/方正小标宋）、正文字体（仿宋）、页边距、落款右对齐位、段间距、表格边框完整性。发现问题立即修正后重新预览。

6. 格式确认后调 Obsidian CLI 打开审阅：
   ```
   "D:\anzhuang\Obsidian\obsidian" open path="[文件名]" vault="Agent知识库"
   ```

#### 3b. 无模板路径（降级）

- 用 python-docx 按「{{USER_SHORT_NAME}}默认排版配置」（页边距 3.7/3.5/2.8/2.6cm，标题黑体 16pt，正文仿宋 14pt，行距 1.5）生成 DOCX
- 同样须走步骤 3a 的第 5 步 OfficeCLI 格式预览

#### 核验提示

附：法条核验 / 信息补充 / 格式核验（模板填充保真 + HTML 预览已确认）

### 步骤4：知识沉淀（软 Hook）
- 定稿交付后，软 Hook 提示："是否需要对比初稿与定稿差异，沉淀到写作习惯？"
- 律师确认 → 读定稿 → 对比差异 → 写 `D:\Agent知识库\写作习惯\`
- 律师跳过 → 不阻塞，正常结束（给了模板时按模板样式，未给模板时按「{{USER_SHORT_NAME}}默认排版配置」）。
