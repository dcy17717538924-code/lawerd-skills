## 流程二：已有案件进展日志的案件的跟进

适用场景：律师提供材料要求跟进已有案件（已有案件进展日志），如补充证据分析、调整诉讼策略、起草补充材料等。

### 步骤 0：MarkItDown / LLM 多模态预处理(原 OCR 预处理,2026-06-05 重构)

律师提供材料后,先进行格式转换。调用策略详见 [[format-conversion]]。

**关键行为：**
- 自动识别材料类型(图片/文档/音频/HTML 等),按策略分发到对应处理路径(MarkItDown / LLM vision)
- 转换完成后,将 Markdown 文本作为后续分析的材料内容

### 步骤 1：加载案件背景

基于 Markdown 输出文本 + 在律师自有知识库("法律"知识库→办案→对应案件文件夹)内查找并读取该案件的案件进展日志,确保完整了解之前的案情、分析要点及历史交互。

**兜底规则：** 如案件进展日志不存在或无法读取——
- 不阻塞流程,基于 Markdown 输出的材料 + 对话上下文继续
- 在 Hooks 中提示律师"未找到该案进展日志,是否需要补充背景信息"

**硬 Hooks：** 将材料理解 + 日志回顾结果向律师汇报，询问以下问题后方可继续——
- 是否有其他材料需要补充（如有材料未提供全）
- 对当前案件阶段的理解是否有偏差
- 识别到的新内容是否准确（如不准确请补充，或告知可忽略）

### 步骤 2：策略/分析产出

得到律师反馈后，进一步给出（或调整后的）诉讼策略、辩论意见、质证意见等分析结果。

策略分析应包含：法律关系定性、请求权基础、举证责任分配、风险识别、建议策略及备选方案。

**硬 Hooks：** 询问律师——
- 分析结果是否有需要调整的地方
- 需要起草何种材料（诉状、申请书、辩论意见、证据目录等）

### 步骤 3：执行起草→归档标准线

根据律师的选择，执行**流程一的步骤 3 至步骤 6**：

> 步骤 3（起草文件）→ 步骤 4（初稿展示与修改）→ 步骤 5（定稿转 DOCX）→ 步骤 6（归档）
