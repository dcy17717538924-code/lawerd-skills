# 法条核验标准工作流

> 本文件由 [[workflow-orchestrator]] skill 加载时读取。兜底工具：`web_search` 多源交叉。

## 5 步工作流

| 步骤 | 操作 | 工具 |
|------|------|------|
| ① 解析 | 法条全称 + 条款号 + 法释编号 / 文号 | 内部解析 |
| ② 找原文 | `web_search("[法条全称] 第 [N] 条 全文")` | web_search |
| ③ 找官方源 | `web_search("[法条全称] site:gov.cn")` 或 `site:court.gov.cn` | web_search |
| ④ 多源交叉 | ≥ 3 个独立源给出一致原文 + 文号 + 实施日期 才确认条款号正确 | 人工+web_search |
| ⑤ 时效性 | `web_search("[法条全称] 废止 OR 修订 2024 2025 2026")` 排除新法/新解释 | web_search |

## 硬证据（3 选 2 才采信）

- ✅ `site:gov.cn` 或 `site:court.gov.cn` 命中
- ✅ ≥ 3 个独立源给出一致原文
- ✅ 法释编号 / 文号命中（如"法释〔2020〕26 号"、"劳部发〔1994〕489 号"）

## 错误信息甄别

- ⚠️ 搜狗 / AI 搜索结果会**编造"2023 年修订"等不存在的版本**
- ⚠️ 没有法释编号 + 没有官方源 + 只有一个源给出原文 → 不可采信
- ⚠️ "将废止"传闻常常滞后 5-10 年（《工资支付暂行规定》"将废止"传了 10+ 年仍未废止），出庭抗辩以"现行有效"为基调
- ⚠️ 商业法务咨询网站（法律快车/华律网/律图）答复可作辅证，但**不能替代官方源**

## 触发场景（自动套用，无需{{USER_SHORT_NAME}}确认）

- 任何法律文书起草涉及法条引用（起诉状/答辩状/代理词/撤裁申请书/质证意见/上诉状等）
- 案件分析涉及法律定性
- {{USER_SHORT_NAME}}说"核验一下 XX 法" / "XX 法还生效吗" / "XX 法第 N 条现行有效吗"

## 输出模板

```markdown
## 法条核验报告 — [法条全称 第 N 条]

- **现行有效性**：✅ 现行有效 / ⚠️ 修订 / ❌ 废止 / ❓ 待复核
- **条款号正确性**：✅ 第 N 条原文与引用一致
- **原文**：（贴法条原文，精确到字符）
- **法释编号 / 文号**：（如有）
- **发布 / 实施日期**：（如有）
- **多源验证**：
  - site:gov.cn 命中：（列具体源）
  - ≥ 3 个独立源给出一致原文：（列源名）
  - 法释编号命中：（是 / 否）
- **时效性风险**：（新法 / 新解释 / 废止传闻，均需说明）
- **文书引用结论**：✅ 正确 / ⚠️ 需修正
```

## 实测参考案例

案例一：最高人民法院关于审理劳动争议案件适用法律问题的解释（一）第 42 条
- 核验结果：法释〔2020〕26 号 + 2020-12-29 发布 + 2021-01-01 施行 + 现行有效
- 核验路径：web_search 多源（14 篇文章一致引用）

案例二：工资支付暂行规定 第 6 条（劳部发〔1994〕489 号）
- 核验结果：1994-12-06 发布 + 1995-01-01 实施 + 现行有效（6 个独立源一致，含 site:gov.cn）
- 时效性风险："将废止"传闻持续多年，至 2026-06 仍未废止
