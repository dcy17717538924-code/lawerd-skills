# 案件卷宗归档（Case Archive Orchestrator）

## 功能概述

把律所纸质卷宗归档沉淀为**可复用 skill**。案件结案后：

1. 自动在案件根建 `00-定卷/` 子目录
2. 按律所 7 大类（诉讼）/ 3 大类（非诉）归类入卷材料
3. 复制并按 `{大类序号}-{文件序号}_{材料名}_p{页码}.{ext}` 命名规范改名
4. 用律所原版模板生成案卷目录.docx 和办案小结（草稿）.docx（1:1 格式）
5. 填写 `诉讼案件已结案表.xlsx`
6. 按 `案卷目录 → 办案小结 → 入卷材料` 顺序合并生成打印版 PDF

## 调用方式

**触发关键词**：
- "归档吧" / "结掉了" / "定稿" / "要装订"
- "律所总表要填"
- 案件根目录出现 `00-定卷/` 前缀的新案件目录（主动检测）

**命令行**：见 `steps/step-05-生成定卷文件.md`

**入口参数**：
- 必填：`-CasePath` 案件根绝对路径（例：`{{CASE_ROOT}}S施玲玲--------离婚`）
- 可选：`-Type` 案件类型（`litigation` 诉讼 / `non-litigation` 非诉，默认 `litigation`）

## 工作流（7 步）

### Step 1：扫描案件根
列出案件根下所有 PDF / DOCX / PNG / JPG 文件，提取元数据：
- 案号 / 法院 / 承办律师（从 PDF 调解书 / 律师函读）
- 立案 / 结案日期（从调解书读）
- 收案日期（从律师函读）

### Step 2：归类 + 复制（按 4 条用户规则）

| 规则 | 说明 |
|---|---|
| **0\uFE0F\u20E3 `3-立案提交/` 优先级最高** | 如 `3-立案提交/` 目录存在，所函、委托书、诉状等材料**优先从此目录取**，因该目录是提交法院的正式签章版，准确度最高。立案提交版与诉讼准备版不一致时，以立案提交版为准。 |
| **1-3 在材料备份文件夹，以图片方式存在** | 大类 1（审理结果）/ 2（委托手续）/ 3（立案材料）优先从 `1-材料备份/` 找 JPG / PNG 图片 |
| **4-5 在案件目录或司法送达等文件夹** | 大类 4（庭审材料）/ 5（保全材料）在 `2-诉讼准备/` / `3-立案提交/` 找 |
| **6 是用户自己截图放在案件文件夹** | 大类 6（签收归还）由用户手动放，skill 自动检测；无则提醒 |

### Step 2.5：四项必查材料

每次归档前必须扫描并处理以下材料：

1. **律师费发票**：优先按文件名搜索 `律师费` / `发票` / `票据`；如在综合扫描 PDF 中，需拆页或作为 PDF 内对应页入卷。
2. **律师聘用合同**：优先按文件名搜索 `聘用合同` / `委托代理合同` / `律师合同` / `委托合同`；图片格式也必须入卷。特别注意：合同经常在 `面谈材料/面谈交接` 目录中，文件名可能是随机压缩图片名；看到图片内容为"委托代理合同"、甲方/乙方、律师费条款、签章页时，应作为律师聘用合同入卷。
3. **诉讼案件已结案表**：使用 `_references/templates/closed_case_table_template.xlsx` 填写，输出到 `00-定卷/诉讼案件已结案表.xlsx`。
4. **散群截图**：优先找文件名含 `散群` / `退群` / `群聊` 的 JPG/PNG；必要时人工看图确认聊天界面含"无法在已退出的群聊中发送消息"等字样。

### Step 3：按律所 7 大类（诉讼）归类

见 `steps/step-03-归类与改名.md`。

### Step 4：按命名规范改名

`{大类序号}-{文件序号}_{材料名}_p{页码}.{ext}`

例：
- `1-1民事调解书_p001.pdf`（大类 1 第 1 个）
- `2-1律师委托合同_p001.jpg`（大类 2 第 1 个）
- `4-1立案通知书_p001.pdf`（大类 4 第 1 个）
- `6散群截图_p001.png`（大类 6 用户手动截图）

### Step 5：填律所原版模板

- 案卷目录.docx（7 行顶部元数据 + 15 行材料清单）
- 办案小结（草稿）.docx（4 行核心表：案由 / 委托人 / 案情 + 体会 / 结果）

案卷目录表格第一列"序号"是**律所目录项目号/大项编号**，不是本类内部自动流水号。必须按材料映射显式提供 `seq`，并以已确认样式为准。连续相同 `seq` 时，第一列必须纵向合并；连续相同 `category` 时，第二列也应纵向合并。注意：律所模板存在合并/不规则单元格，不能只用 `row.cells[0]` 粗暴填写；应使用物理单元格和 `scripts/fix_catalog_sequence.py` 修正，材料超过模板默认 15 行时必须扩展表格行。

### Step 6：填写诉讼案件已结案表

生成：

`00-定卷/诉讼案件已结案表.xlsx`

字段来自案卷目录和律师补充信息。能自动填的先自动填；无法确认的字段保留空白或 `无`，不得编造。

### Step 7：生成打印版合并 PDF

归档完成后，默认再生成：

`00-定卷/打印版-案卷目录-办案小结-入卷材料.pdf`

合并顺序固定为：

1. `案卷目录.docx`
2. `办案小结(草稿).docx`
3. `入卷材料/` 下全部 PDF / DOCX / JPG / PNG，按文件名排序

说明：DOCX 转 PDF 优先使用本机 Microsoft Word；图片转 PDF 使用 Pillow；PDF 合并使用 `pypdf`。如本机 Word 不可用，应保留前两份 DOCX，并提示律师人工打印或安装转换环境。

## 命名规范

| 段 | 规则 | 例 |
|---|------|-----|
| 大类序号 | 1-7（诉讼）/ 1-3（非诉） | 1=审理结果，6=签收 |
| 文件序号 | 同类同种材料从 1 起 | 2-1 / 2-2 委托合同 |
| 材料名 | 用户口语化 | 身份证 / 律师合同 |
| 页码 | `_p001` = PDF 起始页（单页默认 001） | _p001 / _p002 |
| 扩展名 | 跟原文件一致 | .pdf .docx .jpg |

## 关键依赖

- Python 3.12+ + `pypdf`（读/合并 PDF）+ `python-docx`（生成 / 填 docx）+ `openpyxl`（填结案表）+ `Pillow`（图片转 PDF）+ `lxml`
- 可选：`PyMuPDF`（修复结构异常的扫描 PDF 后再合并打印版）
- 律所原版 docx 模板（在 `_references/templates/`）
- 案件根路径（`{{CASE_ROOT}}...`）

## 已知坑

- **案卷目录合并单元格**：律所案卷目录模板"序号/类别"列存在纵向合并，必须使用 `scripts/fix_catalog_sequence.py` 按显式 `seq` 和 `category` 写入物理单元格并设置 `vMerge`，不要用 `row.cells` 直接覆盖。
- **OCR 限制**：如果材料是扫描 PDF（无文字层），`pypdf.extract_text()` 读不出来。**回退方案**：用 matrix MCP 视觉 API（`read_pdf_vision.py`），或用 computer use 截屏 OCR
- **案号格式**：不同律所案号格式可能不同（沪 0106 民初 vs (2025) 沪 0106 民初）。skill 不强校验，提取到的字符串直接填
- **WPS 路径**：案件路径有 WPS 云盘同步（`D:\wpsyunpan\229601413\WPS云盘\...`）和本地副本（`D:\案件管理\...`）两份。skill 跑在 WPS 云盘那份（用户偏好）

## 不适用

- 不是律所纸质卷宗归档（企业行政档案 / 财务凭证）
- 不是结案装订归档，仅需记录案件阶段进展
- 不是案例研究报告（对外发表）— 走 `case-study-report`
