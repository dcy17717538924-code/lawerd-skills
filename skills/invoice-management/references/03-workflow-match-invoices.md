# 流程三：凑票

> {{USER_SHORT_NAME}}说"凑XX元发票"时，从 `未使用/` 选取组合覆盖目标金额，移入日期文件夹。

## 前置条件

- `发票统计表.xlsx` 存在且含"未使用"状态的发票
- Python + openpyxl 已安装

## 步骤

### 1. 读取可用发票

从 `发票统计表.xlsx` 读取"状态=未使用"的行，获取：文件名、用途类型、金额。

### 2. 金额组合

调用 `scripts/match_invoices.py`：

```powershell
& "{{CODEX_PYTHON}}" scripts/match_invoices.py --target {目标金额} --xlsx "{发票根目录}\发票统计表.xlsx" --unused-dir "{发票根目录}\未使用"
```

组合策略（贪心）：
1. 按金额降序排列可用发票
2. 从大到小选取，累计金额 ≥ 目标时停止
3. 最后一张可能超出目标（允许 ±15% 偏差）
4. 若库存不足覆盖目标 → 汇报差额，询问是否接受部分凑票

### 3. 创建日期文件夹

在 `{发票根目录}` 下创建：

```
{YYYY-MM-DD}+{合计金额}/
```

如：`2026-06-09+2150.98/`

### 4. 移入选中发票

```python
shutil.move(f"{未使用}/{文件名}", f"{日期文件夹}/{文件名}")
```

### 5. 更新统计表

选中的发票行：
- 状态 → "已使用"
- 使用日期 → 当日日期
- 使用备注 → {{USER_SHORT_NAME}}提供的用途说明（如"案件差旅费"）

### 6. 汇报

```text
📋 凑票完成 — 目标 ¥{目标金额}
选中 {N} 张，合计 ¥{合计金额}

| 用途 | 金额 |
|------|------|
| 交通费 | 1050.00 |
| 餐饮费 | 218.00 |
| ... | ... |

📁 已移入 {日期}+{合计金额}/
📊 统计表已更新，剩余可用 ¥{剩余金额}
```

## 边缘情况

- 库存总金额不足目标 → 列出可用总额，询问是否接受
- 多张同金额发票 → 优先选最早的（按序号）
- {{USER_SHORT_NAME}}指定用途类型（"凑2000元交通费"）→ 只从该类型中选
