# 案件进度归档 Skill

## 你的工作方式

本技能的核心职责是**维护一份跨会话可追溯的案件进展日志**，确保每次恢复讨论时上下文不断裂。

**核心原则**：
- 每次归档产出两条核心记录：**案件进展日志**（主档案，按`客户名称-案件进展日志`命名）+ 当阶段的**定稿文件**
- 日志内容必须精确，不虚构、不推测，仅基于当前对话中已发生的事实
- 原始证据以**容量瘦身后的版本**归档，归档前**提醒用户使用WPS等工具对原始证据（PDF、图片等）进行容量压缩/瘦身后再提供**。IMA不支持直接上传ZIP压缩包。
- 同时在日志中须提炼**证据摘要**——提炼对话中已讨论过的每份材料的关键要点

### 归档路径简示

> 金山文档云盘 → 案件管理/客户文件夹/agent/ ← 存档位置

## 归档结构（三级目录体系）

> 金山文档云盘 → 案件管理 → [一级] 客户文件夹 → [二级] 事务类型 / agent → [三级] 具体事项

### 一级文件夹：客户

**命名规则**：

| 客户类型 | 格式 | 示例 |
|---------|------|------|
| **个人客户** | `[首字母][姓名]--------[案由]` | `W吴比--------劳动纠纷` |
| **常法客户-诉讼** | `[首字母][名称]-----[客户类型]` | `L良工阀门-----常年顾问` |
| **常法客户-非诉讼** | `[首字母][名称]-----[客户类型]` | 同上，非诉事项在二级区分 |

> 首字母：取客户名称的第一个汉字的拼音首字母大写。如"吴比"取"W"，"良工阀门"取"L"

### 二级文件夹：事务类型 / agent

每个客户文件夹下，默认建立 **`agent/`** 文件夹，专用于存放粽宝的阶段性归档内容（案件进展日志、分析草案、中间产出等）。

| 客户类型 | 二级结构 | 示例 |
|---------|---------|------|
| **所有客户（通用）** | 客户文件夹下自动包含 **`agent/`** | `W吴比--------劳动纠纷/agent/` |
| 个人客户（诉讼） | `agent/` + 其他用户自建的子文件夹并行 | 见示例 |
| 常法客户（诉讼） | 同个人客户 | — |
| 常法客户（非诉讼） | `agent/` + `/非诉讼事务/[类型]-[事项名]（YYYY-MM）/` 并行 | `/非诉讼事务/出函-收房告知书（2026-05）/` |

> **agent 文件夹规则**：在首次归档时检查是否存在，如不存在则自动创建。log/定稿文件均存放于此。存档的原始材料（证据、裁判文书等）仍按客户习惯放入 `1-材料备份/`、`2-诉讼准备/` 等其他用户子文件夹。

### 三级文件夹：具体事项
仅常法非诉讼使用，见上。

## 强制流程（5步）

### Step 1：触发与识别

当用户说以下任何一句或案件有阶段性进展时，执行本技能：

- "先搁置"、"放一放"、"结掉"、"归档吧"、"出了定稿"、"阶段性完成"
- 或在案件代理进展（开庭、收到判决、立案等）后主动执行

首先识别案件要素：

| 要素 | 来源 |
|------|------|
| 当事人姓名 | 对话历史 / 案件日志 |
| 案由 | 对话历史 |
| 客户类型 | 个人客户 / 常法客户（诉讼/非诉讼） |
| 文件夹前缀 | 姓名首字母大写 |
| 沟通次数 | 已有日志的最大次数+1 |

### Step 2：待办确认

查看上次日志的"待办事项"，确认哪些已完成、哪些需延期或搁置。

**规则**：
- 待办状态用符号标识：[x] 已关闭 / [ ] 待办 / [-] 延后
- 已完成的待办，在日志中保留并标记为 ✅
- 未完成的待办，在日志中保留并标记为 ⏸

**首次归档（新人新案）**：无待办确认，跳过此步，直接进入Step 3。

### Step 3：整理本阶段内容

| 字段 | 填写要求 |
|------|---------|
| **事件** | 用一句话总结本阶段做了什么，或案件发生了什么进展 |
| **处理** | 列出具体行动（用1. 2. 3.） |
| **产出** | 列出本阶段产生的所有文件 |
| **关键发现** | 本案独有的重要事实、法律风险、对方策略等 |
| **调整/决策** | 涉及策略方向变更或重要决策，必须记录 |
| **下一步** | 明确后续计划 |
| **待办** | 格式：`- [ ] ▶ 待办事项描述` |

**提醒原始证据归档**：如果本阶段有用户上传但未入库的原始证据（PDF/图片），提醒用户做容量瘦身后发回归档。

### Step 4：文件操作与元数据管理

1. 检查目标客户文件夹下是否存在 `agent/` 子文件夹，如不存在则调用 kdocs create_folder 创建。
2. 写入案件进展日志文件（模板见 `references/template.md`，首次归档时读取填充）至 `agent/` 文件夹内。
3. 将定稿文件上传到 `agent/` 文件夹（如定稿文件需额外存到其他用户子文件夹，同步复一份到对应位置）。
4. 移入文件并告知用户（如果同文件名存在旧版，告知用户手动删除旧版）。

### Step 5：输出与归档告知

最后输出告知：

> "案件进展日志已更新，下次聊这个案子时直接说'接着某某案'，我会从金山文档云盘调日志继续。"

## 跨会话续接机制

当用户说"接着之前那个案子"或提及某个已归档案件的当事人名时：

1. **检索金山文档云盘**：使用 kdocs 的 search_files / list_files 工具在云盘 `案件管理/` 文件夹下搜索匹配的客户文件夹，然后在其 `agent/` 子文件夹下查找 `*案件进展日志*` 文件。
2. **读取日志**：全文读取最新版本的日志，掌握案件全貌。
3. **定位最新进展**：从最后一条记录开始续接。
4. **确认语境**：向用户简要复述案件状态和待办，确认后继续工作。

## 工作流示例

### 示例1：个人客户案件搁置归档

> 用户："先搁置吧，后面客户要起诉再和我说。"

1. 识别案件要素 → 检查是否有待办 → 整理本阶段内容
2. 检查/创建 agent/ 文件夹
3. 更新案件进展日志到 agent/ 目录
4. 归档定稿文件到 agent/ 目录
5. 告知完成

### 示例2：常法客户非诉讼事务归档

> 用户："出了定稿，把这个版本收起来吧。"

1. 判断为常法客户+非诉讼事务
2. 检查/创建 agent/ 文件夹
3. 路径：案件管理/L良工阀门-----常年顾问/agent/
4. 将案件进展日志写入 agent/
5. 常法服务日志写入客户文件夹一级目录
6. 归档定稿文件（存 agent/，如有需要同步复至其他位置）
7. 告知完成

### 示例3：跨会话续接

> 用户："吴比那个案子，我明天开庭，帮我再看一眼。"

1. 检索金山文档云盘 → 案件管理/W吴比--------劳动纠纷/agent/W吴比案件进展日志.md
2. 读取日志 → 复述案情和待办 → 确认后继续工作
