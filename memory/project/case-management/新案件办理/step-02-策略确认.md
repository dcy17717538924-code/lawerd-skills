# step-02：策略确认

## 目的

基于律师对案件简报的反馈，产出诉讼策略分析报告，含策略五要素。

## 输入

- step-01 产出的案件简报
- 律师对硬Hook 的回复

## 输出

- 策略分析报告（含五要素）

## 前置条件

- step-01 的硬Hook 已完成
- 律师已回复确认事项

## 执行

1. 读取律师对 step-01 硬Hook 的回复
2. `IF` 律师有未回复的问题 `THEN` 执行软Hook（提醒一次），`IF` 仍不回复 `THEN` 跳过
3. 基于回复，产出策略分析报告

策略分析报告须包含以下五要素：
- **法律关系定性**：本案涉及何种法律关系
- **请求权基础**：原告/我方主张的法律依据
- **举证责任分配**：各方需证明的事实
- **风险识别**：信息缺失点、逻辑漏洞、对方可能的抗辩路径
- **建议策略**：首选方案及备选方案

4. 核验引用的法条有效性

## 异常处理

- `IF` 律师对策略报告提出异议 `THEN` 按异议调整后重新产出，重复不超过 3 轮
- `IF` 法条核验失败 `THEN` 标注"法条未经核验，请律师确认"

## Hook

- **软Hook**：就 step-01 中律师未回复的问题提醒一次。`IF` 仍未回复 `THEN` 跳过，不阻塞流程
- **硬Hook**：请律师确认诉讼策略
