# step-05：归档

## 目的

将定稿文件及案件进展日志归档至案件管理目录。

## 输入

- step-04 产出的定稿 DOCX 文件路径
- 案件进展日志文本

## 输出

- 已归档的文件（远程路径确认）

## 前置条件

- step-04 已完成，定稿 DOCX 已生成
- 目标归档路径可写入

## 执行

1. 调用 `run_skill("case-progress-archive")`，传入定稿文件路径和案件进展日志
2. 归档路径规则：
   - `IF` 在 `案件管理\` 下找到匹配当前案件的客户名称文件夹 `THEN` 归档至该文件夹
   - `IF` 未找到匹配文件夹 `THEN` 触发硬Hook

## 异常处理

- `IF` 归档路径无法访问 `THEN` 提示"归档路径不可用，请手动指定"
- `IF` 文件写入失败 `THEN` 提示重试

## Hook

- **硬Hook**：`IF` 未找到匹配的客户文件夹 `THEN` 请律师告知该案对应的文件夹命名（客户名称），确认后归档
- **软Hook**：归档完成后提示——"是否需要将本案的争议要点沉淀为专题知识？" → `run_skill("manage-legal-knowledge")`
- **软Hook**：归档完成后提示——"是否需要更新案件进展日志？"
