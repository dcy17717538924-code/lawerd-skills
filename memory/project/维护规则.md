## 1. 同步规则：本地 → workspace

### 什么时候需要同步
- 本地 skill 的 SKILL.md / steps/ / references/ / scripts/ 有实质内容更新
- 本地 memory/global/ 有新增或修改
- 本地 skill 的 CHANGELOG.md 有新版本记录
- 新增自有 skill（记得先更新 skill-ownership.md 白名单）

### 什么时候不需要同步
- 仅本地个人信息变化（姓名、微信、律所等已在 personalization.yaml 中，由 apply-personalization.py 处理）
- 本地 `.reasonix` 运行时生成的文件（sessions/、.obsidian/、config.json 等）
- 第三方 skill 的本地化适配（如 author 字段被 apply-personalization 替换为本名——**这是 bug，应恢复第三方原作者**）


### 署名保护黑名单（这些行**不脱敏**）

匹配以下模式的行**整行跳过**，不做任何替换：

- `author:` / `license:`（YAML frontmatter 或 body 中）
- `Copyright` / `©`
- `作者：` / `许可证：`（中文署名行，包括 `--` 块、CHANGELOG blockquote、body）

LICENSE / LICENSE.txt / LICENSE.md 文件**整文件跳过**。

### 为什么署名不脱敏
自有技能的 `author` 和 `作者：` 是知识产权声明，发布时保留真实姓名。脱敏只针对路径、联系方式、日常行文中出现的个人信息，不针对署名。

## 4. Skill 署名格式规范

### Frontmatter（YAML 头）
```yaml
---
name: skill-name
description: |
  本技能应在用户需要[场景]时使用。
  不要用于：[负向条件]。
---
```
**只允许 `name` 和 `description`。** 禁止 `author` / `version` / `license` 出现在 YAML 头中。

### 文件末尾署名
```markdown
--
- 作者：杜重阳律师（微信Dcylawer8888）
- 版本：X.Y.Z
- 许可证：MIT
```

### CHANGELOG.md 顶部署名
```markdown
> 作者：杜重阳律师（微信Dcylawer8888）
> 许可证：MIT
```

## 6. 安装流程（lawerd-install.md 自动执行）

1. `git clone` 获取仓库
2. 环境自检（参照 `memory/global/tech-environment.md`：检测→缺则安装→验证）
3. `profile-wizard.py` 收集个人信息 → 生成 `personalization.yaml`

5. 部署：`cp skills/* ~/.reasonix/skills/` + `cp memory/global/* ~/.reasonix/memory/global/`
6. 项目记忆不通过 `cp` 部署——AG 读取 `memory/project/` 后通过 `remember` 工具逐条写入
